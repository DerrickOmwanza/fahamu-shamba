┌─────────────────────────────────────────────────────────────────────────────┐
│                          DATA FLOW EXAMPLE                                  │
└─────────────────────────────────────────────────────────────────────────────┘

REGISTRATION FLOW:
  1. Farmer fills form in Farmer Profile Dashboard
  2. POST /api/farmer-profile/register
  3. Data saved to farmer_profiles table
  4. Activity logged to farmer_activity_logs
  5. Response includes farmer_id

RETRIEVAL FLOW:
  1. Dashboard calls GET /api/farmer-profile
  2. Backend queries farmer_profiles table
  3. Returns list with pagination
  4. Frontend displays in table

STATISTICS FLOW:
  1. Admin views /admin/dashboard
  2. GET /api/admin/dashboard
  3. Counts from both farmers and farmer_profiles
  4. Returns combined statistics
  5. Shows in admin dashboard UI

✅ Main Dashboard ← Fetches from /api/farmer-profile
✅ Admin Dashboard ← Gets combined count from /api/admin/dashboard
✅ Farmer Profile Dashboard ← Saves to farmer_profiles table
✅ All systems ← Share same database instance
✅ Activity Tracking ← Logs all changes to farmer_activity_logs
✅ Admin Control ← Can verify and manage profiles via admin routes
