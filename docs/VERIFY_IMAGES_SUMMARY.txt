WHERE IMAGES ARE STORED
═════════════════════════════════════════════════════════════════════════════

Images stored in database as base64:

LOCATION:
  File: backend/fahamu_shamba.db
  Table: farmer_profiles
  Column: passport_photo_url

FORMAT:
  data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAAAQABAAD/2wBDAAgGBgcGBQgHB...

SIZE:
  Each photo: ~130 KB in database
  (from ~100 KB original image + 33% base64 overhead)

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: No photos showing in list
Solution: 
  1. Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  2. Check database: SELECT COUNT(*) FROM farmer_profiles WHERE passport_photo_url IS NOT NULL;
  3. If 0, register farmer with new photo

Problem: Photos not uploading
Solution:
  1. Check file size < 5 MB
  2. Check file type is JPG, PNG, or GIF
  3. Check browser console (F12) for errors
  4. Check server logs for error messages

Problem: Database columns don't exist
Solution:
  Run: node backend/migrate-add-photo-columns.js

Problem: Can't find database file
Solution:
  File location: backend/fahamu_shamba.db
  Verify with: ls -la backend/fahamu_shamba.db
